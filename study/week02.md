# 1. API
### api에는 크게 두종류가 있다.
1. 읿반적인 API
- 서로 다른 어플리케이션이 서로 소통하는데 사용되는 인터페이스
2. RESTful API
- 자원을 표현하고 HTTP 메서드를 사용하여 상태를 전달하는 API

-> 안의 구조를 몰라도 서로 다른 프로그램끼리 정보를 주고 받을수있도록 하는 일종의 통로

-> 사용자는 안에 로직을 알 필요가 전혀 없음. 

http 환경에선 주로 rest api를 사용하여 url이나 사용자의 요청에 정보를 담아서 서버로 전송한다.

API URL의 구성 

ex) GET http://localhost:8080/api/students

1. GET: http 매서드
2. http: 프로토콣
3. localhost: 도메인
4. 8080: 포트번호
5. api.students: 엔드포인트


# 2. Gradle
### 오픈소스 빌드 자동화 도구

- 프로젝트의 컴파, 테스트, 패키징, 배포등 여러 기능 수행

### 가장 중요한 부분: Build.gradle

- Spring을 하다보면 다른건 별로 안들어가지만 build.gradle은 자주 들어감
- 의존성, 플러그인 설정 등 빌드에 대한 모든 기능 정의
- 현재 작업 폴더의 java version, 플러그인, 의존성 등등의 정보를 확인 할 수있고, 변경 가능


# 3. MySQL
### 
MySQL은 전 세계에서 가장 널리 사용되는 오픈 소스 관계형 데이터베이스 관리 시스템(RDBMS). 

데이터를 표(Table) 형태로 정리하여 저장하고, 이를 효율적으로 관리할 수 있게 해주는 소프트웨어.

- 여러 RDBMS가 있지만 mySql은 오라클에서 인수하여 오랜 기간동안 개발자들이 사용해온 가장 안정적인 DB
- 설정이 간편하고 성능이 검증되어 있어, 소규모 프로젝트부터 대규모 서비스까지 폭넓게 사용가능.
