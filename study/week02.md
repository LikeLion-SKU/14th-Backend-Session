# API 정리
## 1. API란?

API(Application Programming Interface)는 **서로 다른 프로그램이 서로 소통할 수 있도록 해주는 인터페이스**이다.

즉, 한 프로그램이 다른 프로그램의 기능을 사용할 수 있도록 정해진 방식으로 요청하고 응답을 주고받는 구조이다.

---

## 2. RESTful API란?

RESTful API는  
웹에서 주로 사용하는 API 방식으로, **자원을 URL로 표현하고 HTTP 메서드를 사용하여 상태를 전달하는 API**이다.

### 특징
- URL로 자원을 표현함
- HTTP 메서드를 사용해서 작업을 구분함

---

## 3. API URL 구조
GET http://localhost:8080/api/students
- GET : HTTP 메서드
- http : 프로토콜
- localhost : 도메인
- 8080 : 포트번호
- /api/students: 엔드포인트

---

## 4. API의 특징
- 서로 다른 시스템 간 통신을 가능하게 함
- 정해진 규칙(인터페이스)을 통해 요청과 응답이 이루어짐
- 내부 구현을 몰라도 기능을 사용할 수 있음
- 주로 HTTP 기반으로 동작 (REST API)

# gradle 정리
## 1. Gradle이란?
Gradle은 **프로젝트를 자동으로 빌드해주는 오픈소스 빌드 자동화 도구**이다.

→ 소스 코드를 실행 가능한 프로그램으로 만들어주는 과정(빌드)을 자동으로 처리한다.

---

## 2. 빌드 도구란?
빌드 도구는  
**코드를 컴파일하고, 테스트하고, 실행 가능한 파일로 만들어주는 도구**이다.

예:
- 컴파일
- 테스트 실행
- 패키징 (jar, war 생성)
- 배포

---

## 3. Gradle의 특징
- 필요한 작업들을 자동으로 처리할 수 있음
- 의존성 라이브러리를 쉽게 관리할 수 있음
- 설정을 코드처럼 작성 

---

## 4. Gradle 주요 파일 
### build.gradle 

- 프로젝트의 **설정 파일**
- 의존성, 플러그인, Java 버전 등을 정의
- 라이브러리를 추가할 때 build.gradle 파일을 자주 수정하게 됨.

# MySQL
## 1. 데이터베이스란?

데이터베이스는  
**전자적으로 저장된 데이터의 집합**이다.

- 여러 사람이 데이터를 공유할 수 있음
- 다양한 형태의 데이터를 저장 가능

---

## 2. 데이터베이스 종류

### 관계형 데이터베이스 (RDB)
- 데이터를 테이블 형태로 저장
- 정해진 구조(스키마)에 따라 저장
- SQL 사용

예: MySQL

---

### 비관계형 데이터베이스 (NoSQL)
- 자유로운 형태로 데이터 저장

예: MongoDB, Redis

---

## 3. MySQL이란?
MySQL은 대표적인 관계형 데이터베이스 관리 시스템(RDBMS)이다.

---

## 4. 테이블 구조
- Column(열) → 속성
- Row(행) → 하나의 데이터

---

## 5. SQL
데이터를 다루는 언어

```sql
-- 사용할 데이터베이스 선택
use likelion; 

-- 현재 데이터베이스에 존재하는 테이블 목록 확인
show tables;

-- student에 저장된 데이터 조회
select * from student; 

-- student 테이블의 구조 확인
desc student;

-- student 테이블에 새로운 데이터 추가
insert into student (name, age) values ('최예윤', 24);
