API
{
    정의: 서로 다른 어플리케이션이 서로 소통하는데 사용되는 인터페이스
    종류: **Rest API**, GraphQL, SOAP, WebSocket... 등
    메소드: HTTP Method 사용(GET, POST, PUT, PATCH, DELETE)
    구성: 메소드 프로토콜://도메인:포트번호/API엔드 포인트 = GET http://localhost:8080/api/hello
    사용 이유: 
        1. 편의성 - 복잡한 방식 없이 API만 알면 사용 가능
        2. 보안성 - 정해진 API를 통해서만 정보를 전달하여 보안성을 높인다
        3. 상호운용성 - 다른 언어나 환경의 프로그래램끼리 편리하게 소통이 가능하다. 

}

Gradle
{
    정의: CI/CD를 위해 아래 작업들을 자동화 시켜 주는 Groovy 기반의 오픈소스 빌드 도구
    사용이유: 다른 빌드 도구보다 Groovy 문법을 사용하여 빠르다. 멀티 프로젝트 빌드 기능을 제공한다
    특징: Groovy 문법의 클로저 기능을 의존성에 활용
    **클로저**:{ } 안에 코드 블록을 담아서 변수처럼 전달하거나 실행할 수 있는 기능
    `build-gradle`에 있는 `dependencies{}` 같은 것
}

MySQL
{
    정의: 오픈소스 관계형 데이터베이스 관리 시스템(RDBMS)
    특징:
        1. 관계형
        2. SQL 사용 - SQL: 데이버 베이스 언어 (= CRUD)
        3. 오픈소스 <--> Oracle은 기능이 많지만 비쌈
    장점:
        1. 스키마 기반
        2. 멀티스레드 기반
        3. 다양한 저장 엔진 선택 가능
        4. 외래 키 사용 가능 - 외래 키: 다른 테이블에서 참조되는 값, 기조 ㄴ키와 다르게 null이나 중복이 허용 
    단점:
        1. 일부 고급 기능 부재 
        2. 트랜잭션 관리와 한계
        3. 대용량 데이터 처리 한계
}
