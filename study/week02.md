API
->서로 다른 어플리케이션이 서로 소통하는데 사용되는 인터페이스
나는 클라이언트와 소통하기 위해서 사용하는 방식으로 이해하고있다.
그 중 REST API가 있는데 우리가 프로젝트를 하면서 지켜야할 규칙
과 비스무리하다.
사전적인 의미는 HTTP를 이용해서 자원을 다루는 API 방식이라고 말하지만
메서드로 구분한다.
5가지가 있는데 GET, POST, PUT, PATCH, DELETE가 있다.

GRADLE
-> 프로젝트를 빌드하고 관리해주는 도구
역할로는 의존성 관리, 빌드, 실행, 테스트 등이 있는데 
쉽게 말해서 컴파일할 때 도와주는 도구라고 이해하고 있다.
그 중 build.gradle을 많이 사용한다.

MYSQL
-> 데이터를 저장하고 관리히는 데이터베이스

우선 데이터베이스를 관리할 때 구조화된 데이터로 관리를 할 것인지,
데이터를 비정규화하여 관리할 것인지 구분된다.
MYSQL같은 데이터베이스는 정규화 데이터베이스로 실생활에서 보이는
액셀이라고 생각하면 된다. 반면 비정규화 데이터베이스는 대표적으로
MongoDB가 있는데 테이블 없이 JSON형태로 저장하는 데이터베이스
라고 생각하면 된다.
MYSQL같은 경우 장점으로는 데이터도 정확하고 구조가 명확해서 안정적이다.
단점으로는 구조 변경이 어렵고 초기 설계가 중요하다.
MongoDB는 컬럼이 필요없어서 유연한 구조로 설계되고 대용량으로 처리가 가능하다.
단점으로는 관계 처리가 약하고 데이터의 일관성이 약하다.
