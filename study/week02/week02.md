# API 
>## API란?
>## 서로 다른 어플리케이션이 서로 소통하는데 사용되는 인터페이스  

># Restful API
>### 본격적으로 스프링에서 사용하게 될 API 형태  
>>### 특징  
>>### 1. Representational State Transfer 아키텍처를 따르는 웹 API
>>### 2. 자원을 표현하고 HTTP 메소드를 사용하여 상태를 전달하는 API

>## API URL 구성   
>>### GET http://localhost:8080/api/hello   
>
>### 다음과 같이 HTTP 메소드, 프로토콜, 도메인, 포트번호, API 엔드포인트로 이루어져있다.

>## HTTP 메소드 5가지
>>### 1.GET : 데이터 조회   
>> 파라미터를 이용해, 데이터를 서버에 전달해줄 수 있다.   
>> 응답코드 : 200(OK)
>>#### ex) GET /api/students?student_num=~~~
>
>>### 2.POST : 데이터 등록
>> 메시지 바디를 이용해, 서버에 데이터를 전달한다.   
>> 응답코드 : 201(Created)   
>
>>### 3.PUT : 데이터 전체 수정
>> 메시지 바디를 이용해, 서버의 데이터의 전체부분을 수정한다.    
>> 응답코드 : 200(OK), 201(Created - 원래 존재X), 204(존재X)
>>#### (주의! : 일부 리소스만 수정 불가능)
> 
>>### 4.PATCH : 데이터 부분 수정
>> 메시지 바디를 이용해, 서버의 데이터를 일부부분 수정한다.   
>> 응답코드 : 200(OK), 201(Created - 원래 존재X), 204(존재X)
>>### PUT VS PATCH
>>#### 바디 내, 필드가 일부만 존재한다면 PUT은 그 필드만 남기고 삭제, PATCH는 그 필드만 수정
> 
>>### 5.DELETE : 데이터 삭제
>> 요청을 받은 해당 엔드포인트의 데이터를 제거한다.   
>> 응답코드 : 200(OK), 204(존재X)

# Gradle

>## Gradle이란?   
>### 오픈 소스 빌드 자동화 도구
>### 프로젝트 컴파일, 테스트, 패키징, 배포 등을 수행

>## Gradle 파일 구조 및 역할
>> .gradle : gradle 버전 별 엔진 및 설정 파일   
>> gradle/wrapper : Gradle을 설치하지 않아도 Gradle task를 실행할 수 있게 함   
>>### build.gradle : 의존성, 플러그인 설정 등 빌드에 대한 모든 기능 정의
>> gradlew & gradlew.bat : Unix & Windows 용 실행 스크립트 (WIN 기준 : gradlew build 빌드)
>> settings.gradle : 프로젝트 설정 파일
> 

>## build.gradle 파일 분석   
>> plugins 블럭 : 환경 구성에 필요한 플러그인 설정    
>> 옵션 (groupId, version, description 등)   
>> java 블럭 : Java 버전 설정   
>> repositories 블럭 : 빌드에 필요한 의존성을 다운받을 저장소   
>> dependencies 블럭 : 프로젝트에 필요한 의존성 설정   
>> task.named('test') 블럭 : JUnit 5 기반의 테스트 실행

# 데이터베이스

>## 데이터베이스란?
>### 전자적으로 저장된 데이터의 집합
>### 여러 사람들과 데이터를 공용할 수 있음
>### 단어, 숫자, 이미지 등 다양한 유형의 데이터가 저장될 수 있음

>## 데이터베이스의 종류
>## 관계형 데이터베이스 VS 비관계형 데이터베이스
>### 구조적인 데이터 저장방식 VS 유연한 데이터 저장 방식   
>### 테이블 형식으로 저장 VS Key-Value, Graph 등 다양한 형식으로 저장   
>### 스키마에 맞게 데이터를 입력 VS 스키마에 따라 데이터를 읽어옴   
>### SQL 언어 사용 VS DBMS마다 각각의 언어 사용   
>### MySQL, PostgreSQL VS MongoDB, Redis


# MySQL
>## MySQL이란?
>### 전세계적으로 가장 널리 사용되고 있는 오픈소스 관계형 데이터베이스   
>### (관계형 데이터베이스 = RDBMS)

>## 테이블이란?
>### 관계형 데이터베이스 안에서 실제 데이터가 저장되는 형태
>> Attribute(열) : 속성(ex: 이름, 학번 등)   
>> Tuple(행) : 가로 줄에 모여있는 값의 집합(ex: 릴레이션 스키마, 릴레이션 인스턴스)   
>> Domain : 데이터 값의 범위   
>> 릴레이션 스키마 : 데이터베이스의 구조가 어떻게 구성되는지에 대한 정보를 담고 있음   
>> 릴레이션 인스턴스 : 스키마에 따라 실제로 저장된 데이터의 집합

>## MySQL 자주 쓰는 명령어, 쿼리 정리
>> mysql -u root -p : mysql에 root 유저로, 비밀번호를 입력해 로그인하겠다는 명령   
> CREATE DATABASE 데이터베이스명; : 데이터베이스 생성   
> USE 데이터베이스명; : 사용할 데이터베이스로 이동   
> SHOW DATABASES; : 모든 데이터베이스 조회   
> SHOW TABLES; : 데이터베이스 내 모든 테이블 조회   
> DESCRIBE 테이블명; : 테이블 정보 조회(구조 확인 가능)   
> INSERT INTO 테이블명 (속성명1, 2....) VALUES (값1, 2...) : 테이블 데이터 삽입   
> SELECT * FROM 테이블명 : 테이블 내 모든 데이터 조회   
> UPDATAE 테이블명 SET 속성명=값 WHERE 조건식(속성명=값); : 조건식에 해당하는 레코드의 정보 수정   
> DELETE FROM 테이블명 WHERE 조건식(속성명=값); : 조건식에 해당하는 레코드 삭제
